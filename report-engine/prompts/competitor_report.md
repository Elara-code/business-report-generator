# 竞品对比分析报告 Prompt

用户输入主题：**{subject}**（可以是"A vs B"或"A vs B vs C"，如"Notion vs Obsidian"、"美团 vs 饿了么"）
报告类型：`competitor`（多产品对比）

## 你的任务

对 **{subject}**涉及的产品做横向对比分析。读者是：
- 想选型某品类的采购方
- 想找差异化的创业者
- 做投资判断的 FA / 投资人

报告需要让他们 5 分钟内看清"谁更适合我 / 谁更有未来"。

## 数据可视化与标签约束（重要！）

为保证 SVG 图表渲染时不溢出 / 重叠，**所有图表的文本标签必须遵守以下长度限制**：
- `categories`（条形图）：**≤ 10 个字符**（"Notion" "Obsidian" 即可，**不要写全名**）
- `axes`（雷达图维度名）：**≤ 6 个字符**（如"易用性""性能""AI""生态"）
- 矩阵图点位名（产品名）：**≤ 8 字**（用产品简称 + 维度后缀，如"Notion.S"）
- 折线图 X 轴时间标签：年份/季度即可
- 画布 9 宫格条目：**每条 ≤ 12 字**
- 漏斗图各阶段名：≤ 8 字

竞品分析中**多产品名混排**最容易超出长度，请用简称并在附录写全名映射。

## 各章节写作要求

### 1. 执行摘要
- 150-250 字
- 3 句话：市场格局、一句话对比结论、推荐谁
- 列 1 张对比表（用 Markdown 表格语法）

### 2. 产品速览
- 每个产品 100-150 字
- 包含：成立时间、定位、核心功能、当前规模（DAU/MAU/ARR 任一）

### 3. 定位对比
- **配 matrix 图**：2x2 象限
- 推荐坐标轴："用户规模 × 商业化深度" 或 "易用性 × 功能专业度"
- 数据格式：`{"x_label": "...", "y_label": "...", "points": [{"name": "产品A", "x": 4.0, "y": 3.5}, ...]}`

### 4. 功能矩阵
- **配 radar 图**：每个产品一条曲线
- 6-8 个维度（功能/性能/生态/AI/协作/集成/移动端/价格友好度）
- 数据格式：
```
{
  "axes": ["维度1", "维度2", ...],
  "series": [
    {"name": "产品A", "values": [4.0, 3.5, ...]},
    {"name": "产品B", "values": [3.0, 4.5, ...]}
  ]
}
```

### 5. 商业模式对比
- 表格 + 文字：每个产品的盈利模式、定价、客单价、ARPU
- **配 canvas 图**：选最代表的产品画商业模式画布（若只有两个产品，对比展示）
- 也可以用 bar 图：客单价对比

### 6. 用户体验对比
- 关键体验维度 3-5 个（上手成本、协作流畅度、响应速度、生态丰富度、视觉设计）
- 每个维度用 1 段话对比

### 7. 增长策略对比
- 表格：获客渠道、关键动作、增长数据
- **配 bar 图**：月活/下载量/收入等可量化的对比

### 8. SWOT 矩阵
- 每个产品一个 SWOT（优势/劣势/机会/威胁），4 个要点
- 用 **配 matrix 图** 或 Markdown 表格呈现
- 数据格式（matrix 适用）：
```
{
  "x_label": "市场吸引力",
  "y_label": "产品竞争力",
  "points": [{"name": "产品A.S", "x": 4.5, "y": 4.0}, ...]
}
```

### 9. 战略总结
- 谁的护城河更深？
- 谁更可能赢未来 3 年？
- 给不同身份的建议：用户选哪个、创业者抄哪个、投资人看哪个
